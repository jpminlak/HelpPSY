** 폴더를 비우고 껍데기만 남긴다.

로컬 저장소 생성
git init

폴더 파일 상세하게 보기
ls -al

원격 저장소 연결
git remote add origin https://github.com/jpminlak/HelpPSY.git

** 이 때 숨겨진 폴더 .git이 생성된다. git의 설정이므로 나중에 이걸 덮어쓰지 않도록 한다.


pull 하기
git pull origin(원격 저장소 별칭) master

** 추가할 파일을 붙여넣는다. 이 때 폴더 통째로 하위 폴더에 그냥 넣는다. 기존 폴더에 덮어쓰지 말자.


추가 파일 대기시키기 (이상한 오류 메시지가 뜨면 다시 한 번 git add . 를 실행해보자!)
git add .

커밋 하기
git commit -m "커밋 내용"

푸시
git push -u origin master

강제 푸시 (푸시가 안되면 쓰자!)
git push -f origin master(현재 브랜치)

===============================

상태 확인
git status

커밋 로그
git log

원격 저장소 확인
git remote -v

* error: failed to push some refs to 오류가 뜰 때
> 원인 - 어떤 파일이 원격 저장소에는 있지만, 내 컴퓨터에 없는 경우 발생.
먼저 push를 하여 원격저장소의 파일을 내 컴퓨터로 가져온 뒤
add, commit, push 하기

파일 생성
touch README.md

파일 삭제
rm README.md
